# 会话报告模板

> 各 Agent 会话结束后按此模板写入 `tasks/shared/session-reports/[角色].md`。
> 普通模式写简版，演练模式写详版。

## 简版（普通模式）

```markdown
## [YYYY-MM-DD HH:MM] [会话目标]

### 执行动作
- [x] 动作1: 描述
- [ ] 动作2: 描述（未完成）

### 改进建议
- [建议描述]

---
```

## 详版（演练模式）

```markdown
## [YYYY-MM-DD HH:MM] [会话目标] 🎭 演练

### 执行动作
- [x] 动作1: 描述
- [ ] 动作2: 描述（未完成）

### 协议合规

> ⚠️ **以下字段为客观事实记录，不是自评**。/rr 复盘时将用这些字段验证协议合规性，不依赖 Agent 的自我判断。

- [ ] `actual_first_output`: AI 本会话实际的第一句输出原文（逐字记录，用于验证标准开场白是否原文）
- [ ] `pre_opening_exists`: 开场白前是否有任何输出（含空行/工具调用/元叙述），如有则记录内容
- [ ] `opening_verbatim_match`: actual_first_output 是否与 standard-openings.md 中对应 Agent 的开场白完全一致（是/否）
- [ ] `iron_door_compliance`: 会话最后一句输出是否仅为"请唤醒 [Agent名]" + 原因（是/否，如否记录实际结尾）
- [ ] 日志记录: 是否写入对应 .log 文件
- [ ] 状态更新: 是否更新 agent-status.md / workers/status.md

### 发现的问题
- [问题描述]（严重程度: P0/P1/P2）

### 改进建议
- [建议描述]

---
```

## 演练模式触发

用户说"唤醒 [Agent名] 演练模式"或"演练模式，唤醒 [Agent名]"时进入演练模式。

## 追加位置

> ⚠️ **追加 ≠ 覆盖**
> - **追加** = 在现有内容之后添加新报告，保留所有原有内容（文件头、历史报告、重置分隔线）
> - **覆盖** = 替换整个文件，丢失所有历史记录
> - **禁止覆盖**。写入报告前必须先读取文件现有内容，在正确位置插入新报告。

新报告**必须**追加在 session-reports 文件的"报告记录"标题正下方，重置分隔线（`## ⏬ 系统重置分隔线`）之前。

```
## 报告记录

> 以下按时间倒序排列，最新报告在最上方。

← 新报告追加在这里（重置分隔线之前）
⚠️ 追加，不要覆盖！

---

## ⏬ 系统重置分隔线 — ...

← 不要追加到这里（重置分隔线之后）
```
