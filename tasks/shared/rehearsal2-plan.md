# Rehearsal 2 演练计划

> 测试 Rehearsal 1 修复效果 + 演练自闭环新功能
> 预计：4 个会话，每个 3-5 分钟

---

## 第一阶段：系统自检（1 个会话）

**新会话**，发送：

```
/help
```

等 AI 回复后，发送：

```
/status
```

等 AI 回复后，发送：

```
/log
```

**检查点**：
- [ ] /help 列出了 /rehearsal-review 命令
- [ ] /status 正确显示 Agent 状态
- [ ] /log 显示了刚才的命令记录（不再是空的！）

---

## 第二阶段：角色切换测试（1 个会话）

**新会话**，发送：

```
唤醒 Planner 演练模式
```

**检查点**：
- [ ] AI 以 Planner 身份回复（说了 Planner 的开场白）
- [ ] AI 不是以"根 Agent"身份回复
- [ ] AI 执行了 Planner 的工作流程（读取 inbox、评估任务等）
- [ ] AI 最后输出"请唤醒 [下一个Agent]"

---

## 第三阶段：待机模式测试（1 个会话）

**新会话**，发送：

```
唤醒 Coordinator 演练模式
```

**检查点**：
- [ ] AI 以 Coordinator 身份回复
- [ ] AI **没有**尝试 Start-Sleep 或任何轮询
- [ ] AI 完成后标记状态为"待机"或"沉睡"
- [ ] AI 输出"请唤醒 [下一个Agent]"

---

## 第四阶段：演练复盘（1 个会话）

**新会话**（就是回到我这里），发送：

```
/rr
```

或者：

```
看看演练结果
```

**检查点**：
- [ ] AI 读取了所有 session-reports
- [ ] AI 输出了问题清单（P0/P1/P2 排序）
- [ ] AI 输出了协议合规汇总
- [ ] 不需要手动复制任何聊天记录！

---

## 总结

| 阶段 | 会话数 | 测试什么 | 发什么 |
|------|--------|---------|--------|
| 1 | 1 | 命令日志写入 | /help → /status → /log |
| 2 | 1 | 角色切换 + 演练模式 | 唤醒 Planner 演练模式 |
| 3 | 1 | 待机模式（无轮询） | 唤醒 Coordinator 演练模式 |
| 4 | 1 | 演练复盘 | /rr |

**共 4 个会话，零手动复制。**
