# Git 分支记录

此文件记录每个 Agent 工作的 Git 分支，防止分支冲突。

## 分支分配规则

- 每个工作 Agent 使用独立的分支
- 分支命名格式: `agent/<agent-id>/<简短描述>`
- 管理员 Agent 使用 `agent/coordinator/` 前缀

## 合并规则

1. 工作 Agent 完成任务后，通知管理员 Agent
2. 管理员 Agent 审查后决定是否合并
3. 合并前确保没有冲突
4. 合并使用 `git merge --no-ff` 保留分支历史

---

## 使用说明

1. 工作 Agent 开始任务前，在此表中登记分支信息
2. 分支创建命令: `git checkout -b agent/xxx/description`
3. 任务完成后，通知管理员进行代码审查和合并
4. 已合并的分支可以删除
